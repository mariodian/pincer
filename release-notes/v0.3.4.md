# v0.3.4

Pincer now stays active even when your Mac sleeps or loses connectivity. This release introduces linked incidents, improved monitoring continuity via the background daemon, and performance improvements to the incident timeline at longer time ranges.

## Highlights

- **Linked incidents** — related incidents can now be connected to each other for better context during outages
- **Daemon reliability** — the background daemon now recovers more cleanly from connectivity drops, sleep/wake cycles, and sync interruptions, keeping your monitoring history intact
- **Faster timeline** — the 24h and 7d incident views now load significantly faster thanks to pre-aggregated data processing
- **Dashboard polish** — tooltip, chart, and color refinements across the dashboard for a cleaner look

## Fixes

- Fixed daemon sync failing to recover after timeouts or disconnects
- Fixed open incidents sometimes being reconstructed incorrectly when switching polling modes
- Fixed threshold and status state getting out of sync during configuration updates
