# v0.3.0

A stability and reliability release focused on memory management, concurrency, and incident data retention.

## Highlights

- **Concurrent health checks** — Pincer now runs up to 10 health checks at the same time for faster status updates across many agents
- **Incident retention** — incident history is now automatically cleaned up based on your configured retention period, preventing unbounded database growth
- **Internal network warning** — agents on private or internal networks now show a notice that they may not be reachable from external networks

## Fixes

- Fixed a memory leak where deleting an agent left behind orphaned status tracking data
- Fixed the status history map growing indefinitely over time
- Fixed incident event history growing without limit (now unified with retention settings)
