# v0.3.8

This release improves reliability around incident recovery and daemon synchronization, with a few polish updates to keep day-to-day monitoring more consistent.

## Highlights

- Incident recovery is more dependable, so ongoing issues are less likely to be lost after app or sync state transitions
- Daemon namespace migration handling is more robust, improving sync continuity in upgraded environments
- Tray status synchronization is more consistent with the latest monitored state

## Fixes

- Fixed edge cases where active incidents could be reconstructed inconsistently
- Fixed minor UI polish in the main app for more consistent visual styling
- Included runtime and dependency maintenance updates to improve overall stability
