# v0.3.6

This release improves daemon synchronization and incident accuracy while adding both auto refresh and manual refresh to make day-to-day monitoring faster.

## Highlights

- Pages now support auto refresh to keep monitoring data up to date in the background
- You can also manually refresh pages on demand to immediately pull the latest data
- Daemon sync is more reliable across namespaces, including improved handling of agent changes and reconnect scenarios
- Internal performance and reliability work improved incident processing and status consistency

## Fixes

- Fixed incident timeline filtering so handoff-only events no longer create noise in views
- Fixed status breakdown accuracy by correcting degraded versus failed count mapping
- Fixed sync edge cases around orphaned agents and timestamp alignment
- Fixed agent cleanup so deleting an agent also removes related records

## Notes

This release also includes substantial internal refactoring, broader automated test coverage, and CI improvements to increase overall stability.
